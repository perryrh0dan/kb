# kb ‚Äî Knowledge Base CLI & MCP Server: Design Document

**Date:** 2026-07-15
**Status:** Approved

---

## Overview

`kb` is a single Go binary that ingests documents from multiple sources, chunks and embeds them, stores them in a local SQLite database with vector search, and exposes them via CLI and MCP server. The primary use case is building a private knowledge base accessible from AI tools like OpenCode.

---

## Architecture

Single binary, monolithic approach. CLI and MCP server share identical code paths for search and ingest. No plugin system ‚Äî new adapters and embedders are Go files implementing internal interfaces.

## Core Interfaces

```go
// Source delivers documents from a data source
type Source interface {
    Documents(ctx context.Context) (<-chan Document, error)
}

// Embedder converts text into vectors
type Embedder interface {
    Embed(ctx context.Context, texts []string) ([][]float32, error)
    Dimensions() int
    ModelName() string
}
```

New adapters implement `Source`. New embedding providers implement `Embedder`. No other code changes needed.

### Embedder Factory

```go
// internal/embedder/embedder.go
func New(cfg config.EmbedderConfig) (Embedder, error) {
    switch cfg.Provider {
    case "openai":
        return openai.New(cfg)
    default:
        return nil, fmt.Errorf("unknown embedder provider: %s", cfg.Provider)
    }
}
```

Adding a new embedder = new subdirectory + new case in the factory switch.

## Data Model

### Document

```go
type Document struct {
    ID          string            // Source URI, e.g. "confluence://ENG/123456" or "file:///abs/path/doc.md"
    Title       string
    Content     string
    ContentHash string            // SHA256 of raw content, used for change detection
    SourceType  string            // "file" | "confluence"
    Metadata    map[string]string // author, url, updated_at, tags, etc.
    IngestedAt  time.Time
}
```

### Database Schema

```sql
CREATE TABLE documents (
    id           TEXT PRIMARY KEY,
    title        TEXT NOT NULL,
    source_type  TEXT NOT NULL,
    content_hash TEXT NOT NULL,
    metadata     TEXT NOT NULL,  -- JSON blob
    ingested_at  DATETIME NOT NULL
);

CREATE TABLE chunks (
    id          TEXT PRIMARY KEY,  -- UUID
    document_id TEXT NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    content     TEXT NOT NULL,
    chunk_index INTEGER NOT NULL,
    embedding   F32_BLOB(3072)     -- sqlite-vec, 3072 dims for text-embedding-3-large
);

CREATE INDEX idx_chunks_document_id ON chunks(document_id);
```

`ON DELETE CASCADE` ensures deleting a document automatically removes all its chunks.

## Ingest Pipeline

### Incremental Sync (default)

```
known_ids = store.GetAllDocumentIDs(source_type)
seen_ids  = {}

for each document from adapter (via channel):
    seen_ids.add(doc.ID)
    hash = SHA256(doc.Content)
    existing = store.GetDocument(doc.ID)

    if existing != nil && existing.ContentHash == hash:
        continue  // unchanged, skip

    chunks = chunker.Split(doc.Content)
    embeddings = embedder.Embed(chunks)
    store.DeleteChunks(doc.ID)      // remove old chunks
    store.UpsertDocument(doc)
    store.SaveChunks(doc.ID, chunks, embeddings)

// Phase 2: prune deleted documents
stale_ids = known_ids - seen_ids
for each stale_id:
    store.DeleteDocument(stale_id)  // cascades to chunks
```

### Full Re-index (`--force`)

Same pipeline but skips the hash check ‚Äî always re-chunks and re-embeds every document.

## Chunker

Recursive Character Splitter (implemented from scratch, ~150 lines):

- Splits on `\n\n`, then `\n`, then `. `, then ` `, then characters ‚Äî in order, until chunks are within size limit
- Configurable `chunk_size` (default: 512 tokens) and `chunk_overlap` (default: 50 tokens)
- Token counting via `pkoukk/tiktoken-go`

## Adapters

### File Adapter

- Walks a directory (optionally recursive)
- Filters by extension (default: `md`, `txt`, `pdf`)
- Document ID: `file:///absolute/path/to/file`
- Metadata: filename, path, file modification time
- PDF text extraction via `dslipak/pdf`

### Confluence Adapter

- Supports both **Cloud** (username + API Token) and **Data Center** (PAT)
- Auth detection: if `pat` is set ‚Üí PAT auth; otherwise `username` + `api_token` ‚Üí Cloud auth
- Fetches all pages in a space via Confluence REST API v2
- Optionally scoped to a single page + its children (`--page <id>`)
- Document ID: `confluence://<space>/<pageID>`
- Metadata: title, author, URL, last modified, space key, labels
- Converts Confluence Storage Format (HTML) to plain text for chunking

---

## Embedder

### OpenAI (initial implementation)

- Model: `text-embedding-3-large` (3072 dimensions)
- Uses `sashabaranov/go-openai`
- Batches embedding requests (max 100 texts per API call)
- Extensible via factory pattern

## CLI Commands

```
kb ingest                                    # sync all configured sources
kb ingest file <path> [flags]               # ingest + register/update source in config
kb ingest confluence --space <KEY> [flags]  # ingest + register/update source in config

  Flags:
    --recursive          walk subdirectories (file only)
    --ext md,txt,pdf     file extensions to include (file only)
    --page <id>          scope to page + children (confluence only)
    --force              full re-index, skip hash check

kb search <query> [flags]
  Flags:
    --limit 10           number of results (default: 10)
    --min-score 0.7      minimum similarity score filter
    --source file|confluence  filter by source type

kb serve                                    # start MCP server (stdio)
kb status                                   # DB stats: doc count, chunk count, last ingests
kb config init                              # create ~/.kb/config.yaml with defaults
```

---

## MCP Server

Runs via **stdio** ‚Äî MCP client starts `kb serve` as subprocess.

Uses `modelcontextprotocol/go-sdk`.

### Exposed Tools

- `search_knowledge_base(query, limit?, min_score?, source?)` ‚Üí array of rich results
- `list_sources()` ‚Üí configured sources with stats
- `get_document(document_id)` ‚Üí full document content + metadata

## Configuration

### `~/.kb/config.yaml`

```yaml
openai:
  api_key: ""         # or KB_OPENAI_API_KEY

confluence:
  base_url: ""
  username: ""        # Cloud
  api_token: ""       # Cloud, or KB_CONFLUENCE_API_TOKEN
  pat: ""             # Data Center, or KB_CONFLUENCE_PAT

db:
  path: ~/.kb/kb.db

embedder:
  provider: openai
  model: text-embedding-3-large

chunker:
  chunk_size: 512
  chunk_overlap: 50

sources:
  - type: file
    path: /home/user/docs
    recursive: true
    extensions: [md, txt, pdf]
  - type: confluence
    space: ENG
```
